# Cold Storage: Projects

Active project context, key decisions, and rationale.
**When to load:** working on a specific project, reviewing scope, picking up where you left off after a gap.

## Decision Log

Use this section to record decisions that aren't obvious from the code or files — especially ones where the "why" will be forgotten.

**[Project name] — [Decision made]**
- Context: what problem prompted this
- Options considered: brief list
- Why this one: the actual reason
- Revisit if: conditions under which this should be reconsidered

**Why this matters:** The decision is easy to see in the outcome. The reasoning isn't. Recording the "why" means you don't re-litigate it from scratch six months later, and you know when conditions have changed enough to warrant revisiting.

## Active Constraints

Things that aren't changing right now, even if they're not ideal — so you don't waste time working around them.

- **[Constraint]** — [why it exists, when it might change]

Example: "Auth system is read-only until Q3 — security review in progress. Don't propose changes that touch auth."

## Naming & Terminology

Project-specific vocabulary that matters for consistency:

- **[Term]** — what it means in this project's context (especially if it differs from the common definition)
